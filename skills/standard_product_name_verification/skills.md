# standard_product_name_verification — 标准商品/服务名称查询

## 功能说明

将任务需要明确的商品或服务名称匹配到国家税收分类编码体系中的标准名称、类型，并返回对应税率。服务会一次返回**多个候选标准商品类型**及其匹配度。

## 何时使用

- 查询某个商品 / 服务 / 货物的"商品类型"、"标准名称"、"税收分类编码名称"
- 确定某商品或服务的适用税率，但使用的是口语化/非标准名称
- 需要确认用户提到的商品在税务开票时应使用的规范名称
- 回答中涉及增值税税率，且需要通过商品名称反查税率

## 关键触发词

`商品名称`、`服务名称`、`标准名称`、`税收分类编码`、`税率查询`、`开票名称`、`货物名称`、`应税服务`、`税目`、`商品编码`

## 调用方式

> 本技能已封装为命令行工具，**大模型直接生成下列 bash 命令交给执行环境运行即可**，无需自行编写 Python。

### 单个查询

```bash
python -m skills.standard_product_name_verification "马粪"
```

默认输出自然语言文本，可直接嵌入回答：

```
「马粪」在税收分类编码体系中的候选匹配：
  - 有机肥料（肥料），税率 9%，匹配度 1.0
  - 动物、植物肥料（肥料），税率 9%，匹配度 0.631
```

### 批量查询（一次进程，多个名称）

```bash
python -m skills.standard_product_name_verification "马粪" "电脑" "理发服务"
```

多条结果之间以空行分隔。

### 查看帮助

```bash
python -m skills.standard_product_name_verification --help
```

## 注意事项

- 商品名称包含空格、引号或特殊字符时**必须用双引号**包裹：`"代驾 服务"`
- 多个名称用空格分隔，每个独立加引号：`"A" "B" "C"`
- 匹配度仅作参考信息带出，供大模型做参考
- 默认输出自然语言文本，适合直接嵌入回答；需要程序化处理再加 `--json`
- 命令必须在项目根目录下执行（即包含 `skills/` 目录的目录）
- 退出码：成功为 `0`；存在请求/解析失败时为 `1`
