# 统一 EXPLORE 动作 + 三层目录树实施方案（v0/v1 版本分离）

## 目录结构

```
reasoner/
  __init__.py               # 空
  v0/
    __init__.py             # 空
    engine.py               # 原 engine.py，导入改为 reasoner.v0.xxx
    prompts.py              # 原 prompts.py，无逻辑修改
    react_agent.py          # 原 react_agent.py，导入改为 reasoner.v0.xxx
    agent_graph.py          # 原 agent_graph.py，导入改为 reasoner.v0.xxx
  v1/
    __init__.py             # 空
    engine.py               # 从 v1.agent_graph 导入，其余逻辑与 v0 一致
    prompts.py              # 从 v0 导入不变 prompt + 定义新 prompt
    react_agent.py          # 核心改造：统一 EXPLORE + 三层目录树
    agent_graph.py          # 从 v0 导入共享类 + 简化版 AgentGraph
```

## 设计决策

- **v0 保持可运行**：仅做导入路径的机械性修改（`reasoner.xxx` -> `reasoner.v0.xxx`），逻辑零修改
- **v1 最大化复用 v0**：共享类（ExploredRegistry、PitfallsRegistry、KnowledgeFragment、RetrievalKnowledgeRegistry）和不变的 prompt 直接从 v0 导入
- **BacktrackIntent 整套机制移除**：已认领目录在三层树中不可见，竞态冲突由 `try_claim` 失败跳过兜底
- **三层目录树**替换原有 `upstream_path` + `available_subdirs` 双段展示
- **统一执行逻辑**：不区分方向，1 个目标原地 continue，多个 spawn 子智能体

将 `engine.py`、`prompts.py`、`react_agent.py`、`agent_graph.py` 复制到 `reasoner/v0/`，修改全部内部导入：

## v1：核心改造

### 1. v1/prompts.py — 三层目录树 + 统一 EXPLORE

从 v0 导入不变的 prompt（CONTENT_ASSESS_PROMPT、ROOT_DISCLOSURE_PROMPT、PATH_CORRECTION_PROMPT 等），仅定义变更的 prompt：

**新 DISCLOSURE_PROMPT / RETRIEVAL_DISCLOSURE_PROMPT**

prompt 上下文从两段变为一段「可探索目录」三层树：

```
## 可探索目录

### 父级可选目录（与 {parent_name} 同级）
- 1_会计处理
- 3_实务操作

### 当前级可选目录（与 {current_name} 同级）
- 2.2_企业所得税
- 2.3_其他税种

### 子级可选目录（{current_name} 的子目录）
- 2.1.1_xxx
- 2.1.2_进项税额
```

导航选项从 3 个简化为 2 个：

```json
// EXPLORE — 统一探索动作
{
  "action": "EXPLORE",
  "targets": ["目录名1", "目录名2"],
  "reasons": ["选择目录1的理由", "选择目录2的理由"],
  "seekings": ["期望在目录1找到什么", "期望在目录2找到什么"],
  "current_summary": "当前已收集信息摘要"
}

// STOP — 终止
{
  "action": "STOP",
  "reason": "终止理由"
}
```

探索策略引导：
> 1. **优先向下** — 子级目录有内容时，优先深入获取细粒度知识
> 2. **必要时横向** — 当前路径知识已充分或不相关时，跳转到当前级同级目录
> 3. **必要时向上** — 需要更广视角时，跳转到父级同级目录

**修改 SUMMARY_PROMPT**：移除 `{intent_results}` 段落。

**不引入 INTENT_RESOLVE_PROMPT**。

### 3. v1/agent_graph.py — 简化版

从 v0 导入共享类：

```python
from reasoner.v0.agent_graph import (
    ExploredRegistry, PitfallsRegistry,
    KnowledgeFragment, RetrievalKnowledgeRegistry,
)
```

AgentGraph 的变更：
- `_standard_pipeline`：直接调汇总，不走 intent 解决
- `_build_evidence_parts`：只返回 agent_parts，不构建 intent_parts
- `_final_summary`：prompt 不含 `{intent_results}`
- `_build_trace_log`：不输出 backtrack_intent 和 intent_resolve 信息
- 删除 `_find_backtrack_intents`、`_resolve_intents`
- 移除 `intent_resolve_results` 属性

其余方法（batched summary、retrieval pipeline、pitfalls check、clean answer 等）与 v0 完全一致，直接复制或通过继承复用。
